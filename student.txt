Q14: explain what is a linked list?
Report:
A **linked list** is a linear data structure where elements, called **nodes**, are stored in non-contiguous memory locations. Each node contains both **data** and a **pointer** (or reference) to the next node in the sequence, forming a chain.

Unlike arrays, linked lists allow efficient **insertions and deletions** at any position without reallocating or shifting elements, since only pointers need updating. However, they lack **random access** — you must traverse from the head node to reach a specific element, making searches slower.

A common use case is implementing **stacks, queues**, or managing dynamic data where frequent insertions/deletions occur. Want a comparison with arrays or details on types (singly vs. doubly linked)?
