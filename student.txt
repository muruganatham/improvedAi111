Q14: explain what is a linked list?
Report:
A **linked list** is a linear data structure where each element (called a **node**) stores data and a reference (or **pointer**) to the next node. Unlike arrays, elements are not stored in contiguous memory locations—instead, each node is dynamically allocated and linked together via these pointers.

**Why it matters:** It allows efficient **insertions and deletions** at any position (O(1) for the head/tail if you have a pointer) since only pointers need updating. However, it lacks direct indexing (access is O(n)) and uses extra memory for pointers.

**Common use cases:**  
- Implementing **stacks, queues, or adjacency lists** in graphs.  
- Dynamic memory allocation (where data size isn't known beforehand).  
- Used in applications like **undo functionality** in software or **browser history**.
