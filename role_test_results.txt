### Q: what is the difference between stack and queue
**Response (4.7s):**
```text
A stack is a linear data structure that follows the Last-In, First-Out (LIFO) principle. This means the last element added is the first one to be removed. Operations are typically called push (add) and pop (remove). Stacks are often used for managing function calls, undo/redo features, or parsing expressions.

A queue, conversely, is a linear data structure that follows the First-In, First-Out (FIFO) principle. The first element added is the first one to be removed. Operations are typically called enqueue (add) and dequeue (remove). Queues are commonly used for task scheduling, managing print jobs, or in breadth-first search algorithms.

The fundamental difference lies in the order of element access and removal: LIFO for stacks and FIFO for queues.
```

### Q: what is the difference between abstract class and interface in java
**Response (4.7s):**
```text
An abstract class can have both abstract and concrete (implemented) methods, constructors, and instance variables. It is used to provide a common base for subclasses, allowing some methods to be defined while others are left for implementation by specific subclasses. A class can only extend one abstract class (single inheritance).

An interface, on the other hand, defines a contract. Before Java 8, it could only contain abstract methods and public static final variables. Since Java 8, it can also include default and static methods. A class can implement multiple interfaces (multiple inheritance of type). Interfaces are used to achieve polymorphism and define a set of behaviors that implementing classes must adhere to.

The main differences are that abstract classes can have state and implemented methods, while interfaces primarily define behavior contracts. A class can extend only one abstract class but implement many interfaces.
```

### Q: explain how binary search tree works
**Response (5.6s):**
```text
A binary search tree is a hierarchical data structure where each node has at most two children. It enforces a specific ordering rule: for any node, all values in its left subtree are less than the node's value, and all values in its right subtree are greater than the node's value. This property is fundamental to its operation.

To find a value, you start at the root. If the target value is less than the current node's value, you traverse to the left child; if greater, you go to the right child. This continues until the value is found or you reach an empty spot, indicating the value is not present. This systematic division of the search space makes operations efficient.

Binary search trees are commonly used for implementing dictionaries or sets, where fast lookups, insertions, and deletions of unique elements are required.
```
